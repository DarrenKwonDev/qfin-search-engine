# qfin-search-engine

## 목적
arXiv에서 사용자의 목적에 맞는 q-fin 논문을 찾고, 핵심 내용을 일목 요연하게 요약해 다른 LLM이 트레이딩 전략/테스팅 코드로 전환할 수 있도록 한다. 이는 개인화된 소규모 논문 추천 시스템으로서, 사용이 누적될수록 맥락적 지식과 검색 속도가 향상되는 점진적 개선을 목표로 한다.

## 기대 동작
- 사용자 목적(시장, 기간, 자산군, 제약, 리스크 성향 등)을 입력으로 받아 논문 후보를 자동 선별한다.
- 논문을 읽고 요약본을 생성한다.
- 요약본은 트레이딩 전략/테스팅 코드로 변환 가능한 구조를 갖춘다.
- 반복 사용 시, 사용자 선호와 컨텍스트를 학습해 검색 정확도와 속도를 개선한다.

## 요약본 포맷(LLM 전달용)
- 논문 메타
  - 제목, 저자, arXiv ID, 공개일
- 문제 정의 및 배경
- 핵심 가설/아이디어
- 데이터
  - 데이터 소스, 기간, 샘플 구성, 피처
- 방법론
  - 모델/알고리즘, 학습/검증 절차, 하이퍼파라미터
- 실험/결과
  - 주요 지표, 비교 기준, 성능 요약
- 트레이딩 전략 관점
  - 진입/청산 규칙, 시그널 생성 방식, 리스크 관리, 거래 비용 가정
- 재현성
  - 코드/데이터 공개 여부, 구현 난이도, 필요한 의존성
- 한계 및 실패 사례
- 확장 아이디어

## 아키텍처 방식
- 입력 의도 정규화: 사용자 목적을 구조화(시장, 기간, 자산군, 제약, 리스크 성향)해 검색 조건으로 변환한다.
- 후보 수집: arXiv에서 q-fin 영역의 논문 메타를 수집하고 중복을 제거한다.
- 선별/랭킹: 키워드 적합도와 정량 기준(기간, 벤치마크, 거래 비용 가정 등)으로 우선순위를 정한다.
- 본문 처리: PDF/본문 텍스트를 추출해 섹션별로 분리한다.
- 구조화 요약: 요약본 포맷에 맞춰 전략 전환이 가능한 정보로 정리한다.
- 피드백 루프: 사용자 피드백과 선택 기록을 축적해 다음 검색/랭킹에 반영한다.

## 점진적 개선 방향
- 사용자 목적과 선호를 축적해 검색 쿼리/필터를 최적화한다.
- 이전 요약본의 피드백을 반영해 요약 구조와 정보 밀도를 개선한다.
- 논문 선정 기준을 정량화(예: 실험 기간, 벤치마크, 거래 비용 가정 등)해 자동 랭킹한다.
